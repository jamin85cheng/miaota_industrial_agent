## 1. 项目概述

### 1.1 项目定位
**面向工业场景的智能监控、异常检测、故障预测与辅助决策系统**

成为工厂运维人员的"数字搭档"，实现：
- 让每一台设备都会"说话"
- 让每一个异常都有"解释"
- 让每一次决策都有"依据"

### 1.2 目标用户
| 用户角色 | 需求 | 使用场景 |
|----------|------|----------|
| **运维工程师** | 实时监控、异常告警、故障诊断 | 日常巡检、异常处理 |
| **工艺工程师** | 趋势分析、参数优化、预测预警 | 工艺调整、预防性维护 |
| **管理人员** | 报表统计、效率分析、决策支持 | 生产管理、绩效考核 |
| **维修人员** | 故障排查、维修指导、备件管理 | 设备维修、故障修复 |

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 采集延迟 | < 5秒 | 从PLC读取到存储 |
| 告警延迟 | < 3秒 | 异常发生到告警产生 |
| API响应 | < 200ms | 95%请求响应时间 |
| 前端首屏 | < 3秒 | 首次 meaningful paint |
| 并发连接 | > 1000 | WebSocket同时连接数 |
| 数据存储 | > 1年 | 原始数据保留时长 |

### 3.2 可用性要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 系统可用性 | 99.9% | 年度停机时间 < 8.76小时 |
| 故障恢复 | < 5分钟 | 自动重启或切换 |
| 数据持久化 | 100% | 已确认数据不丢失 |

### 3.3 安全要求

| 要求 | 说明 |
|------|------|
| 传输加密 | TLS 1.3 |
| 存储加密 | 敏感配置加密存储 |
| 访问控制 | 最小权限原则 |
| 审计追踪 | 所有操作可追溯 |
| 数据脱敏 | 日志中敏感信息脱敏 |

### 4.2 数据流

#### 4.2.1 采集数据流

```
┌─────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ PLC │───▶│ 采集器  │───▶│ 预处理  │───▶│ 规则引擎│───▶│ 时序存储│
└─────┘    └─────────┘    └─────────┘    └────┬────┘    └─────────┘
                                              │
                                              ▼ (如果触发)
                                         ┌─────────┐
                                         │ 告警生成│
                                         └────┬────┘
                                              │
                         ┌────────────────────┼────────────────────┐
                         ▼                    ▼                    ▼
                   ┌─────────┐         ┌─────────┐          ┌─────────┐
                   │WebSocket│         │通知服务 │          │审计日志 │
                   │推送    │         │(飞书等)│          │记录    │
                   └─────────┘         └─────────┘          └─────────┘
```

**流程说明**:
1. **PLC** → 原始数据（寄存器值）
2. **采集器** → 按配置间隔读取，添加时间戳
3. **预处理** → 清洗、格式转换、质量标记
4. **规则引擎** → 实时评估，触发告警
5. **时序存储** → 写入InfluxDB/SQLite
6. **告警** → 通过WebSocket/通知推送

```
                            └──────┬──────┘

```

**流程说明**:
1. **用户输入** → 故障症状描述
2. **RAG检索** → 从向量库找相关文档
3. **构建Prompt** → 症状 + 上下文 + 知识
4. **LLM推理** → 生成诊断结果
5. **结构化输出** → 根本原因、建议、备件
6. **多路输出** → 展示/存储/审计

### 6.2 数据模型

**告警 (Alert)**:
```json
{
  "alert_id": "ALERT_001",
  "rule_id": "RULE_001",
  "rule_name": "缺氧异常",
  "severity": "critical",
  "message": "溶解氧浓度低于 2.0 mg/L",
  "timestamp": "2024-01-15T14:32:10Z",
  "acknowledged": false,
  "suggested_actions": ["增加曝气量", "检查曝气盘"]
}
```

**诊断结果 (Diagnosis)**:
```json
{
  "diagnosis_id": "DIAG_001",
  "root_cause": "曝气量不足",
  "confidence": 0.85,
  "possible_causes": ["曝气盘堵塞", "风机故障"],
  "suggested_actions": ["清洗曝气盘", "检查风机"],
  "spare_parts": ["曝气盘", "风机滤网"]
}
```
